# 如何扩 AI经理的知识和职责

扩 AI经理 时，不要把所有变化都塞进 prompt。

应该按下面这条顺序判断：

## 1. 先加知识条目

如果新增的是：

- 平台规则
- 接入方式
- 术语解释
- 角色边界
- 推荐工作流
- 常见问题回答

优先往 `knowledge/manager/*.md` 里加知识条目。

这类内容本质是“可被检索的稳定知识”，不是运行时状态。

## 2. 再考虑加工具

如果新增的是：

- 要查实时员工状态
- 要查实时任务状态
- 要发起任务分派
- 要处理审批
- 要切到直连

那就不该只写进知识库，而应该新增或增强经理工具。

工具负责“查实时世界”和“改实时世界”。
知识库负责“解释系统”和“指导工作流”。

## 3. 最后才改协议

如果经理能力扩张会影响：

- 员工注册
- 任务生命周期
- 审批流
- 手动/自动切换规则
- 工作区声明方式

那就不只是补知识或补工具了，而是要更新协议和文档。

## 一条简单判断规则

- 稳定事实 -> 知识库
- 实时状态 -> 工具
- 系统行为变化 -> 协议

经理回答“别的 Agent 怎么接入平台”属于知识库问题。
经理回答“Codex Main 现在在做什么”属于工具问题。
经理新增“跨设备群协作任务”如果要落状态流转，就属于协议问题。
